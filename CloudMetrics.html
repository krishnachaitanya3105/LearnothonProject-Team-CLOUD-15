💡 Infrastructure Insights

🚀 LA Benefits Most from CDN
Despite initial connection overhead, Los Angeles shows 32% faster total load time with CDN+LB (0.443s vs 0.651s). Request reduction (3→2) and optimization outweigh connection latency costs. Best for North American users.
⚠️ Beijing Performance Critical
CDN+LB adds +26% latency penalty in Beijing (1.685s vs 1.342s). The load balancer IP routes through suboptimal paths or additional inspection points. Not recommended for China traffic without dedicated China CDN.
🔄 Mumbai Stable Either Way
Mumbai shows minimal variance between architectures. Slight improvements in page weight (7KB→6KB) and requests (3→2) offset minor TTFB increases. Safe to use CDN without significant impact.
📉 Universal Security Score Drop
CDN+LB configuration shows universal Best Practices drop (100→78 for Mumbai/LA, 96→78 for Beijing). Likely due to missing security headers, SSL configuration issues, or load balancer response handling differences.
🎯 Consistent Core Web Vitals
Both architectures maintain perfect 0.0 CLS and 0ms TBT across all regions, indicating frontend code stability regardless of backend infrastructure choice.
⚡ TTFB vs Total Time Trade-off
CDN+LB shows higher TTFB in all regions but lower total time in LA due to request optimization. Mumbai shows minimal difference, Beijing shows degradation in both metrics.
